# Tech Stack & LGA Golden Rules

- **Stack:** React (Vite), TailwindCSS, Supabase (PostgreSQL, RPCs, Triggers), Sanity CMS.
- **Strict i18n:** ALL user-facing text must use the `t("key")` function. If you create new text, IMMEDIATELY update the 5 JSON files in `src/i18n/` (es, en, ca, fr, it). NO HARDCODING.
- **Premium Design (Glassmorphism):** We use an "Ice Glass" style for light mode (`bg-blue-50/40 backdrop-blur-2xl border-white/70 shadow-[0_4px_16px_-4px_rgba(59,130,246,0.05)]`) and deep dark mode (`dark:bg-gray-900/70 dark:border-white/10`).
- **Defensive Programming:** When calling Supabase or RPCs, assume they might fail or not exist yet. Always use try/catch and fallbacks.
- **Product Philosophy:** Minimize user clicks. Drive Network Effects (if a guest has no account, prompt creation and use `sessionStorage` for temporary state).

# Working Method
Before implementing:
1. Identify relevant files and read their imports/dependencies.
2. Understand the current behavior.
3. Plan the minimal change required.

During implementation:
- Change the smallest surface possible.
- Maintain existing contracts.

After implementation:
- Validate behavior.
- Summarize changes clearly.

# When Uncertain
